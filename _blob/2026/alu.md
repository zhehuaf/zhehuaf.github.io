ALU（算术逻辑单元）是 CPU 中负责计算的核心部件，执行算术运算（加减）和逻辑运算（与或非）。

## 加法电路

### 基本逻辑门
- **XOR门**: 计算加法的低位结果
- **AND门**: 计算进位信号

### 半加器
处理两个单比特输入 A、B，输出和与进位：
```
A XOR B → Sum
A AND B → Carry
```

### 全加器
处理三个输入（A、B、前位进位），输出新的和与进位。可由两个半加器和一个OR门实现。

### 多位加法器
将多个全加器串联，形成8位、16位等加法器。进位逐级传播会产生延迟，现代处理器使用超前进位加法器提高速度。

## 指令集架构

### CISC vs RISC

**CISC（复杂指令集）**
- 单条指令功能强大
- 例子：`ADD [A], 5` （取数+运算+存回一步完成）
- 代表：x86架构

**RISC（精简指令集）**
- 指令简单，长度固定
- 需要多步完成复杂操作：
  ```
  LDR R1, [A]    ; 加载到寄存器
  ADD R1, R1, #5 ; 运算
  STR R1, [A]    ; 写回内存
  ```
- 代表：ARM架构

### 编译过程

同一份C代码在不同架构下生成不同汇编：

**C代码：**
```c
int add(int a, int b) {
    return a + b;
}
```

**x86汇编：**
```
add:
    mov eax, edi
    add eax, esi
    ret
```

**ARM汇编：**
```
add:
    add r0, r0, r1
    bx lr
```

**RISC-V汇编：**
```
add:
    add a0, a0, a1
    ret
```

## 文件格式

### ELF文件结构
- **.text节**: 存放机器指令
- **.data节**: 存放全局变量
- **.bss节**: 存放未初始化的全局变量（只记录大小，不占实际空间）
- **.rodata节**: 只读数据，比如字符串常量
- **符号表和重定位信息**: 符号表（函数名、变量名），重定位信息（用于链接时地址修正）

### 编译链接流程
C代码 → 汇编代码 → 目标文件(.o) → 可执行文件
链接器将多个目标文件合并，修正地址引用，生成最终可执行程序。
